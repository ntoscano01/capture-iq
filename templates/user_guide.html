Getting Started

Default Credentials (Development)
  • Username: admin
  • Password: admin123

Important: Change these credentials in production!

First Steps
  1. Log in with your credentials
  2. Go to Dashboard to see project pipeline
  3. Create a project or browse existing ones
  4. If you're a Capture Manager, access Capture Planning

Key Features

For Everyone
  • Dashboard: View project pipeline and recent activity
  • Projects: Manage SBIR project proposals
  • Topics: Browse and search SBIR solicitation topics
For Admins
  • Users: Manage user accounts, reset passwords, unlock accounts
  • Settings: Configure application settings
  • Role Elevation: Promote users to Capture Manager
For Capture Managers
  • Capture Planning: Create and manage capture plans
  • Team Management: Add team members to capture plans
  • Project Linking: Connect projects to capture plans
  • Progress Tracking: Track capture plan status through stages

Common Tasks

Elevating a User to Capture Manager
  1. Log in as admin
  2. Go to Admin → Users
  3. Find the user you want to promote
  4. Click the ✓ (shield icon) button next to their name
  5. Confirm the action in the dialog
  6. User is now a Capture Manager!
Creating a Project
  1. Go to Projects in the navigation
  2. Click Create Project
  3. Fill in project details:
    • Project name
    • SBIR topic/solicitation
    • Phase (Phase I or Phase II)
    • Due date (optional)
  4. Click Create Project
Sharing a Project
  1. Open the project you want to share
  2. Scroll to Team Members section
  3. Click Add Team Member
  4. Select a user from the dropdown
  5. Choose their role:
    • Editor: Can edit project and upload files
    • Viewer: Can view and comment only
  6. Click Add
Leaving a Comment
  1. Open a project
  2. Scroll to Team Comments section
  3. Type your comment in the text box
  4. Click Post Comment
  5. Team members will be notified automatically

Capture Planning (For Capture Managers)

Creating a Capture Plan
  1. Go to Capture Planning in navigation
  2. Click Create Capture Plan
  3. Fill in capture plan details:
    • Plan Name: e.g., "Company X Proposal"
    • Customer Name: The target customer
    • Confidence Level: High, Medium, or Low
    • Proposal Due Date: When proposal is due
    • Win Probability: Estimated chance of winning (optional)
  4. Click Create Plan
Managing Team Members on Plans
  1. Open a capture plan
  2. Find Team Members section
  3. Click Add Team Member
  4. Select a user to add to the plan
  5. To remove, click the ✕ button next to their name
Linking Projects to Capture Plans
  1. Open a capture plan
  2. Scroll to Linked Projects section
  3. Select a project from the dropdown
  4. Click Link Project
  5. Project is now linked to the capture plan
Capture Plan Stages

Plans automatically display in stages to track progress:

  • 🕐 Pre-Release: Before RFP is released
  • 📧 RFP Released: RFP has been released
  • ✓ Proposal Submitted: Our proposal submitted
  • 🏆 Won: We won the contract
  • ✗ Lost: We didn't win

Collaboration Features

Team Comments

Keep your team in sync with comments on projects:

  • Comments appear immediately for all team members
  • All team members are notified when someone comments
  • You can delete your own comments (owners can delete any comment)
  • Use comments to ask questions, share feedback, and confirm milestones
Linking External Documents

Link proposal documents from SharePoint or Google Drive:

  1. Open the project
  2. Scroll to Shared Documents section
  3. Click Link Document
  4. Fill in:
    • Document Source: SharePoint, Google Drive, or Other URL
    • Document URL: Full link to your document
    • Title (optional): Friendly name (e.g., "Tech Proposal V1")
  5. Click Link Document

Tip: Use descriptive titles and include version numbers for drafts

Notifications

Stay updated with automatic notifications:

  • You're notified when someone adds you to a project
  • You're notified when someone comments on your project
  • Check your notification bell in the sidebar for unread count
  • Notifications update automatically (refresh not needed)

Project Roles Explained

Owner

Full control over the project

Can:

  • ✅ Edit project details
  • ✅ Upload/delete files
  • ✅ Manage team members
  • ✅ Delete project
  • ✅ Delete any comment
Editor

Active contributor to project

Can:

  • ✅ Edit project details
  • ✅ Upload files
  • ✅ Comment
  • ✅ Delete own comments
Viewer

Reviewer or observer

Can:

  • ✅ View project
  • ✅ View files
  • ✅ Comment
  • ✅ Delete own comments

Common Workflows

Scenario 1: Getting Feedback on a Draft
  1. You create a project for a new SBIR proposal
  2. You upload the technical proposal draft
  3. You share with Sarah (Editor) and Bob (Viewer)
  4. Sarah uploads her cost proposal section
  5. Sarah comments: "See cost section, needs review"
  6. Bob comments: "Budget looks reasonable"
  7. You see notifications and update the draft
  8. You upload the new version
Scenario 2: Team Proposal Development
  1. Alice creates project "Phase 1 Proposal"
  2. Alice invites Bob (Editor), Charlie (Editor), Diana (Viewer)
  3. Bob uploads technical volume
  4. Charlie uploads management plan
  5. All team members see progress in one place
  6. Notifications alert everyone when new files appear
  7. Team comments document all decisions
Scenario 3: Multi-Week Development
  1. Project created with due date
  2. Team members see progress toward deadline
  3. Comments document decisions made
  4. Activity log shows what changed when
  5. Notifications keep everyone in sync
  6. Linked documents stay updated in SharePoint/Drive

Tips & Tricks

Pro Tips
  • Use descriptive project names that include solicitation name and phase
  • Assign clear roles — Editors for active contributors, Viewers for reviewers
  • When commenting, reference specific files and page numbers
  • Link documents early — don't wait until the last minute
  • Check notifications daily to stay on top of team activity
  • Use comments to celebrate milestones and maintain team morale
Things to Know
  • Deleting a comment is permanent — even owners can't undo it
  • Removing a team member doesn't delete their comments
  • Linked documents are links only — deleting them from SharePoint/Drive removes access
  • Projects are private by default — only invited members can access
  • Always save documents to their original location (SharePoint/Drive) for version control
