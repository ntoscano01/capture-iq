1. Getting Started

Launching the App

Start CaptureIQ from your terminal:

cd sbir-pipeline
python3 launcher.py

The app opens automatically in your browser at http://127.0.0.1:5000. Keep the terminal window open — closing it stops the app.

Default Credentials
  • Username: admin
  • Password: admin123

Change these immediately via Admin → Users after first login.

First Steps
  1. Log in and change the admin password
  2. Create user accounts for your team (Admin → Users)
  3. Assign roles: regular users, Capture Managers, or Admins
  4. Go to Pipeline → Ingest to pull in SBIR solicitation data
  5. Set up an Automated Ingest Job to keep data fresh daily
  6. Browse topics, create projects, and start capture planning

3. Dashboard

The Dashboard is your home base. It shows a live snapshot of pipeline health and team activity.

Pipeline Metrics
  • Total Solicitations: All SBIR topics ingested into the system
  • Favorites: Topics you've starred for follow-up
  • Nominated: Topics marked for active pursuit
  • Pipeline Value: Estimated total award value across all tracked topics
Capture Metrics (Capture Managers)
  • Active capture plans broken out by stage
  • Win probability distribution
  • Plans approaching proposal deadline
Recent Projects

Lists your most recently updated projects with quick-access links to each.

4. Pipeline (Topics & Ingest)

Browsing Topics

Go to Pipeline → Topics to browse all ingested SBIR solicitations.

  • Filter by agency, phase, keyword, or status (Favorite, Nominated, Passed)
  • Click any topic to view full detail including description, eligibility, and award info
  • Star (⭐) a topic to add it to your favorites
  • Mark topics as Nominate (pursuing) or Pass (not pursuing)
  • Export filtered results to CSV
Manual Ingest

Go to Pipeline → Ingest to manually pull fresh data from any source:

  • SBIR.gov: Official SBIR/STTR solicitation database
  • Navy SBIR: Navy-specific topics from the Navy portal
  • DoD SBIR/STTR: Department of Defense solicitations

Each ingest shows records added, updated, and any errors in the log below.

5. Automated Ingest Jobs

Automated Jobs let you schedule recurring data pulls so your solicitation data stays current without manual effort. Go to Pipeline → Auto Jobs to manage them.

Creating a Job
  1. Click New Job
  2. Select a Data Source (SBIR.gov, Navy, or DoD SBIR/STTR)
  3. Choose a Schedule Type:
    • Daily — runs every day (or selected days) at the configured time
    • Weekly — runs on a specific day of the week
    • Monthly — runs on the 1st of every month
  4. Set the Run Time in 24-hour format (e.g., 06:00 for 6 AM). Times are in your local machine time.
  5. Click Create Job
Editing a Job
  1. Click the pencil icon on any job row
  2. Adjust the source, schedule, days, or run time
  3. Click Save Changes
Run Now

Click the Run Now button on any job to trigger it immediately without waiting for the scheduled time. The page will refresh and you'll see "Last Run" and "Last Status" update after the job completes.

Pausing & Resuming

Use the / button to pause or resume any job. Paused jobs are shown in grey and will not run until resumed.

How the scheduler works: The scheduler checks for due jobs every 60 seconds while the app is running. It uses your local machine time, so set run times in your local timezone. The app must be running via python3 launcher.py for scheduled jobs to fire. Watch the terminal for [Scheduler] Tick messages to confirm it is active.
Troubleshooting Automated Jobs

Most common cause: The app was started with python app.py instead of python3 launcher.py.

  • Stop the app (Ctrl+C) and restart with: python3 launcher.py
  • Watch the terminal — you should see [Scheduler] Automated ingest scheduler started (local time). on startup
  • Every minute you should see [Scheduler] Tick — local time: ...
  • If you see two Tick lines per minute, that's the Flask debug reloader (harmless but confirms the guard is working)

  • Check the terminal for [Scheduler] Tick lines — if missing, the scheduler thread isn't running. Restart with python3 launcher.py.
  • Verify the run time in the job matches your local clock exactly (HH:MM in 24-hour format). The scheduler checks every 60 seconds so you have a 60-second window each minute.
  • If the Tick is showing but jobs aren't firing, check that the job is Active (not Paused) and the run_time matches the current local time.

  • Hard-refresh the page with Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows) to clear browser cache
  • Make sure you're running the latest version of the code — restart the app after any updates
  • Open your browser's developer console (F12 → Console tab) and look for any JavaScript errors

  • Check your internet connection — the ingestors fetch data from external SBIR portals
  • The external source may be temporarily down or returning unexpected data; try again in a few minutes
  • Use Run Now to trigger immediately and watch the terminal for a full error traceback
  • If errors persist, go to Pipeline → Ingest and try a manual ingest for the same source — the error message there will be more descriptive

The scheduler runs inside the CaptureIQ app process. If the Mac restarts, the app needs to be restarted manually.

  • Open Terminal and run python3 launcher.py from the sbir-pipeline folder
  • The scheduled jobs will resume automatically once the app is running
  • For fully automatic startup, configure a macOS Login Item or launchd plist — ask your system administrator

6. Projects

Creating a Project
  1. Go to Projects in the navigation
  2. Click Create Project
  3. Fill in:
    • Project Name (e.g., "Phase I Proposal – Navy Topic N25-001")
    • SBIR Topic / Solicitation
    • Phase (Phase I or Phase II)
    • Due Date (optional)
  4. Click Create Project — you are automatically added as the owner
Adding Team Members
  1. Open the project and go to the Team tab
  2. Click Add Team Member
  3. Select a user from the dropdown
  4. Choose their role: Lead, Editor, or Viewer
  5. Click Add
Uploading Files

Upload proposal documents directly to a project (technical volume, cost section, SOW, etc.):

  1. Open the project
  2. Scroll to Files
  3. Click Upload File and select a file
  4. Team members can view and download uploaded files
Linking External Documents

Link to documents stored in SharePoint or Google Drive:

  1. Open the project → Scroll to Shared Documents
  2. Click Link Document
  3. Select the source (SharePoint, Google Drive, Other URL) and paste the link
  4. Add a title like "Tech Proposal V2"
Tracking Progress

Update the project Stage to reflect where it is in the proposal lifecycle:

  • Planning → Writing → Review → Submitted → Won / Lost / No Bid

7. Tasks

Tasks is a cross-project to-do system for tracking individual action items across the team.

Creating a Task
  1. Go to Tasks in the navigation
  2. Click New Task
  3. Fill in:
    • Title — brief description of the action item
    • Project (optional) — link to a specific project
    • Assigned To — which team member is responsible
    • Priority — Low, Medium, or High
    • Due Date
  4. Click Create Task
Managing Tasks
  • Filter by assignee, status, priority, or project using the filter bar
  • Update task status (Open → In Progress → Complete) from the task row
  • Overdue tasks are highlighted in red
  • Edit or delete tasks using the action buttons

8. Capture Planning Capture Manager

Creating a Capture Plan
  1. Go to Capture → Plans
  2. Click Create Capture Plan
  3. Fill in:
    • Plan Name
    • Customer / Agency
    • Confidence Level: High, Medium, or Low
    • Proposal Due Date
    • Win Probability % (optional)
  4. Click Create Plan
Managing Team Members on a Plan
  1. Open a capture plan → find the Team Members section
  2. Click Add Team Member and select a user
  3. Remove members using the ✕ button
Linking Projects to a Plan
  1. Open the capture plan → scroll to Linked Projects
  2. Click Link Project
  3. Select an existing project from the modal
Capture Plan Stages
  • Pre-Release — Before the RFP drops
  • RFP Released — RFP is out; active proposal writing
  • Proposal Submitted — Submitted and awaiting decision
  • Won — Contract awarded
  • Lost — Not selected

9. Key Contacts Capture Manager

Track agency contacts, contracting officers, and POCs associated with each capture plan.

Adding a Contact to a Capture Plan
  1. Open a capture plan
  2. Scroll to the Key Contacts section
  3. Click Add Contact
  4. Fill in name, email, phone, agency, title, and any notes
  5. Click Add Contact
All Key Contacts View

Go to Capture → All Contacts to see every contact across all plans in one place. You can:

  • Filter by name, agency, or associated plan
  • Sort by any column
  • See last-contacted date at a glance

10. Scoring & Rankings Capture Manager

Score and rank SBIR opportunities objectively against weighted criteria so you can prioritize where to invest proposal effort.

Scoring a Project
  1. Open a project or capture plan
  2. Go to the Scoring tab
  3. Add scoring criteria (e.g., Technical Fit, Win Probability, Revenue Potential) with weights
  4. Score each criterion from 1–10
  5. The weighted score is calculated automatically
Rankings

Go to a capture plan's Rankings tab to see all linked projects sorted by total weighted score, helping you quickly see which opportunities are highest priority.

11. Collaboration Features

Team Comments
  • Post comments on any project to keep discussion centralized
  • All team members are notified when someone posts
  • Delete your own comments (owners can delete any comment)
Notifications
  • You're notified when added to a project or someone comments
  • The notification bell in the sidebar shows the unread count
  • Click any notification to go directly to the relevant project

12. Admin Panel Admin Only

User Management
  • Create new user accounts with username, email, and initial role
  • Assign roles: User, Capture Manager, or Admin
  • Reset passwords and unlock locked accounts
  • Deactivate accounts without deleting them
Database Backup
  1. Go to Admin → Database
  2. Click Download Backup — downloads captureiq.db to your computer
  3. Keep regular backups in a safe location (e.g., shared drive)
  4. The Backup History table logs every backup with timestamp and file size
Database Import / Restore
Warning: Restoring from a backup replaces all current data. This cannot be undone. Download a fresh backup before restoring.
  1. Go to Admin → Database → Import / Restore
  2. Click Choose File and select a .db backup file
  3. Click Import & Restore
  4. Restart the application after restore for changes to take full effect
Audit Log

Go to Admin → Audit Log to see a full history of user actions: logins, project changes, user edits, and more.

App Settings

Configure SMTP email settings so the app can send password-reset emails and notifications to your team.

14. Status Indicators

User Status
  • Active — User can log in
  • Inactive — Account disabled
  • Locked — Too many failed login attempts
Job Status
  • Active — Job will run on schedule
  • Paused — Job suspended
  • Success — Last run completed
  • Error — Last run failed
Confidence Levels
  • High — Strong win probability
  • Medium — Moderate win probability
  • Low — Lower win probability
Topic Status
  • Favorite — Starred for follow-up
  • Nominated — Actively pursuing
  • Passed — Not pursuing

15. Tips & Tricks

  • Use descriptive project names: include solicitation number and phase (e.g., "N25-001 Phase I")
  • Schedule ingest jobs to run early morning (e.g., 05:00) before your team starts work
  • Use Run Now after creating a job to confirm it works before waiting for the scheduled time
  • Download a DB backup before any major data change or restore operation
  • Use capture plan scoring to prioritize bids objectively — add weighted criteria and score each opportunity
  • Link external documents early — don't wait until submission day
  • Assign clear roles: Editors for active contributors, Viewers for reviewers/oversight
  • Check the terminal when troubleshooting — all scheduler and ingest activity is logged there

16. Troubleshooting

  • Check username spelling (case-sensitive)
  • Check Caps Lock is off
  • Account may be locked after repeated failures — ask an admin to unlock it

  1. Log in as admin → Admin → Users
  2. Find the locked user and click the unlock 🔓 button

  • You need to be elevated to the Capture Manager role
  • Ask your admin to update your role in Admin → Users
  • After role change, refresh the page

  • Hard-refresh the page (Cmd+Shift+R / Ctrl+Shift+R)
  • Make sure you're logged in as the correct user
  • Projects are only visible to the owner and invited team members

  • The source portal may not have published new solicitations since the last ingest
  • Check your internet connection
  • Try ingesting from a different source to confirm connectivity
  • Check the ingest log for any error messages

Another process is already using port 5000. Kill it:

lsof -ti :5000 | xargs kill -9

Then restart with python3 launcher.py.
